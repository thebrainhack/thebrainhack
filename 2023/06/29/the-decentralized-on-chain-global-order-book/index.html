去中心化的链上全球订单簿

如果卖家可以列出待售产品并立即传播到多个市场怎么办？想象一下 类似MLS（多重列表服务）的功能，但适用于任何资产和公众。这就是Ordinal Lock的本质 ，它是一种利用 sCrypt开发的比特币脚本，sCrypt 是用于在 BSV 上编写智能合约的 JavaScript/TypeScript 框架。

基于 RelayX的订单锁（与 RUN 代币具有基本相同的功能），序数锁允许卖家将其代币锁定在区块链上的比特币脚本中，其中解锁代币的唯一方法是向卖家支付固定金额的 Bitcoin SV 到他们指定的地址。卖家可以随时取消此链上列表。

双重支出 是不可能的，因为脚本的安全性是由矿工在第 1 层上强制执行的。任何第二次尝试解锁脚本都会失败，因此比在两次交易中与代币分开发送硬币或处理 OTC 安全得多托管人，从而引入交易对手风险。之前我写过关于 链上、无需信任的点对点交易如何成为商业的理想选择，但从涵盖PSBT 的技术角度来看 。

PSBT 本质上是私有的。这些肯定有用，具体取决于卖家希望如何与客户打交道。然而，公开交易有很大的好处。

公开记录的完整销售数据给了市场信心。如果每个人都知道交易发生的时间和金额，那么每个参与者都可以利用相同的信息库继续前进。

市场讨厌不确定性。

市场的信心和安全会吸引流动性。此外，当卖家知道以前的销售价格时，他们会更放心地列出待售资产。与中心化交易所相反，中心化交易所的交易数据存储在私人服务器上，这为 市场操纵提供了机会 ，例如扣留某些交易、欺骗以及激励 抢占 自己的客户。

BTC的 Ordinals 市场迫切需要像全球订单簿这样的东西，但他们甚至没有意识到他们所要求的不仅可以通过不受阻碍的比特币脚本实现，而且已经实现。

BTC 的用户体验显然正在朝着这一目标迈进， 在Ordinals Wallet、 Unisat和 Ordswap 等市场上实现了 可互操作的钱包。例如，使用 Unisat 钱包，我可以登录所有三个市场并在每个交易所列出相同的资产。

然而，这是一种糟糕的用户体验，因为他们必须为他们想要列出的每个交易所执行相同的操作。在技术方面，如果在一个市场上购买该资产，则该资产在列出该资产的所有其他市场上都会无效，这会在这些市场上产生错误和更差的用户体验。此外，如果新的交易所上线并具有新功能或为所列出的此类资产吸引流动性，如果用户无法快速手动列出，则可能会错过潜在的销售机会。

全球订单簿不存在这些问题。列表将立即同步到所有市场，新的交易会立即启动订单和销售数据。或者，市场可以策划他们展示的待售商品，但每个人都可以观察公共账本并得出相同的结论。

想象一下，您只在 Zillow 上列出待售房屋（而不是在 Redfin、MLS、Realtor.com 等上）。请注意，由于 MLS 是私有的，并且虽然高效，但确实存在信息同步问题。即使确实发生了降价或销售，过时的列表可能仍会保留在某些网站上。使用比特币，信息可以立即传播，通知各方销售已完成。因此，市场也可以立即对该信息做出反应。这就是 超比特币化 经济的样子。
