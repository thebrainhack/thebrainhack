BTC+LN比IPv4更糟糕

BTC 每秒最多可以处理 7 笔交易 (TPS)。闪电网络 (LN) 有望解决该问题。理论上，正常的交易只会发生在闪电网络中，但偶尔会在 BTC 链上结算。

除了闪电网络不起作用之外，只需一点点数学就能揭示 BTC 不可避免的扩容问题：

大约在 7 TPS 时，世界上每个人一生只能进行一次链上交易。

没有那么自由和光明的未来。

这是另一种看待这个问题的方法：

本质上，每笔需要最终确定性或问责制（其中之一，而不是两者）的交易都需要在 Layer-1 上进行结算。

争论一个普通人一生只需要进行一次这样的交易，甚至比认为 IPv4 足以满足整个世界的想法还要糟糕得多。

大约有 30 亿个 IPv4 地址可用（理论上有 43 亿个）。最初，人们说这绰绰有余。然后他们很快意识到这还不够。但随后他们争辩说，共享 IP 地址将成为解决方案。但他们不知道（大多数人今天仍然不知道）他们不必要地把数字人类关进了一个不仅太小而且环境完美的笼子里，这会招致穷困潦倒的虐待。

原因：IPv4 本质上使 IP 到 IP 交易变得不切实际。结果，IPv4 的局限性成为当今集中式 Web 的罪魁祸首，因为共享 IP 地址使集中化显得权宜之计，几乎是必要的。

缺乏在 TCP/IP 级别集成的 Internet 本地支付方法是另一个罪魁祸首。

那是历史。

今天，我们终于有了针对上述两个问题的解决方案，特别是 IPv6 取代 IPv4，以及在 TCP/IP 上与 IPv6 集成以支持 IP 到 IP 交易的可扩展公共区块链。

但是人们现在声称链上 5-7 TPS 就足够了，因为一切都可以转移到一个不安全的“公社”上来共享有限的链上资源！

对于了解科技史和经济学的人来说，这听起来应该很熟悉。

人们很容易被充斥着意识形态叙事的模糊想法所说服，却没有真正了解人类社会和经济的任何现实、其规模、活动水平以及深远的潜力。

如今，仅信用卡一项，全球平均就有大约 10,000 TPS。那只是信用卡。支持闪电网络的人错误地认为大多数信用卡交易不需要确定性或问责制，因此可以在闪电网络上处理。这种假设是基于对人类现实的无知。

这是现实：

(1) 今天的绝大多数交易，包括信用卡交易，都将受益于最终性和问责制，因此需要及时结算，即使不是立即结算。这些应该都在 L1 上。

(2) 不需要最终性和问责制的交易今天甚至都没有发生，因为我们没有提供足够的系统来实现它们。但是同时具有 L1 和 L2 的比特币 (BSV) 将使它们成为可能。

真正的比特币解决方案不仅是改进现有的交易，而且还使人们能够发明今天不可能甚至无法想象的新型交易。

两个不同世界的图片：

BTC：L1 TPS 5-7，承诺闪电网络上的 TPS 稍高一些，这是不安全的，并且即使在原则上也具有未经证实的可扩展性，更不用说在现实中了（因为需要多跳的网络本质上是 NP-hard to scale）

BSV：L1 TPS 以十亿计，L2 TPS 无限使用支付渠道和覆盖网络。

数十亿的 L1 TPS 怎么可能？

以上是关于未来而不是现在。但是 10 亿 TPS 并不是没有任何道理的随机数。

从长远来看，十亿TPS既是必要的也是可行的。

必要性请看链接文章：区块链Layer-1的必要可扩展性。

对于可行性：

由于并行化和无限块大小，BSV 上的 L1 TPS 没有理论上的限制，但只有工程限制。

首先是主要节点之间的数据传输速率。在 10 亿 TPS 时，它需要介于 250GB/s 到 1TB/s 之间的数据传输速率。这将是一个巨大的挑战，但它完全符合当前的技术参数。供您参考，迄今为止建立的互联网记录数据传输速度超过 40 TB/s。不可否认，这是在受控条件下完成的。但它显示了我们目前面临的潜在终点边界。

其次，这种规模的大块传输将需要在主要节点之间建立 IPv6 组播作为一个高度连接的组播组。猜猜哪个区块链专注于 IPv6 及其在 TCP/IP 级别与区块链的集成？当然不是那些认为区块链是在互联网应用层构建和运行的应用程序的人。

但这些只是基本要求。大区块工程为实际交易的接收和处理、内存池管理、微服务等开辟了丰富的技术创新领域。猜猜在过去的七年里，谁在这个领域投入了大量资金？当然不是那些只想处理几兆字节块的人。
