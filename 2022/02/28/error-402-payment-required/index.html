重新激活“HTTP错误代码402”，释放Web3.0的潜力

是时候夺回互联网了！

在这个互联网充斥着广告和恶意软件的时代，个人数据和网络冲浪历史被收集并贩卖，用以偿付网络上大量免费提供的信息。我们身边很少有人会记得，如果不是由广告公司接管了互联网，互联网可能会是什么样。

但是，如果我告诉你还有另一条路可选呢？

Web 3.0

当人们这些天在谈论Web 3.0时，他们大多是指NFT。这些人很可能是那些命名了Web 2.0的人，这（Web 3.0）只是他们对以太坊的可编程互联网计算机的流行称谓：一个被炒作却从未实现的梦想。但这些人可能错了，因为Web 3.0实际上关乎的是为数据进行支付的能力，或者说是让数据得以赚钱的能力。这是一枚双面硬币，两面均有裨益。这意味着你作为数据内容的创造者，可以直接将他人对你数据的访问进行变现，削弱中间人和门户网站（如Facebook和Google）的权力，但这也意味着作为用户，你可以直接为数据、数据搜索和带宽进行付费，这代表着互联网本身基于广告的盈利模式将划上句号。为什么？

让我们回到早期的网络时代。对于你们这些年轻人，需要先向你们厘清一下网络（web）和互联网（internet）之间的区别。互联网是一个宽泛的术语，描述的是建立在（互联网协议）TCP/IP之上的各种网络。“网络”实际上只是一个TCP/IP之上的应用协议或覆盖协议，该协议即HTTP（超文本传输协议）。HTTP是网络浏览器使用的协议，通过该协议可以传输网络数据或网页。

在网络的支持下，整个10万亿美元的电子商务产业才得以建立。这是一个肇始于小型线上书店和二手垃圾线上拍卖的产业。在亚马逊和E-bay之前，对于像IBM、Sun、甲骨文、网威、思科、英特尔、苹果和微软这样的大公司来说，互联网只不过是一个数字信息亭。(有些公司比其它公司更成功。）但是，将互联网从业余爱好者的幻想转变为成熟商业领域的真正原因是电子商务的诞生（将网络购物作为主要的销售渠道）。

这一切都要归功于一家名为网景的小型初创公司，他们创造了一款小型软件，只不过是用于HTTP的渲染和查看，当时被称为“导航器”。

抛开历史的教训不谈，虽然今天互联网上的广告是如此普遍，以至于人们甚至都不会去注意它们，但重要的是我们要记得，自网络诞生之后十年左右的时间里——再后来网络开始变得流行——广告这种东西是不存在的。数据主机（或网络服务器）没有办法通过其存储或发送数据的服务来获取报酬。因此，没有什么动力去进行数据索引，因此，我们现在所说的无处不在的网络搜索服务当时仍有待发展。在早期，有人尝试过运行一个基于“注册”的索引服务，作一个物理类比的话，就相当于是黄页，但收效甚微。网站和服务器的动态性太强，任何注册服务都无法跟上步伐。

然后，大救星谷歌突然降临了。

谷歌的创始人计划去抓取网络数据，浏览每一个超链接，并实时、持续地绘制互联网地图，并使用这个数据库来支撑起一款搜索引擎，从而免费帮助人们找到他们感兴趣的网站。又因为他们无法从这项服务中获得报酬，所以他们把宝押在：如果他们能够让上网的每个人都使用他们的搜索引擎，那么最终，他们将能够在自己的搜索网站上向广告投放者出售网站“地产”。无处不在的互联网门户业务由此诞生。黄页模式败北，而杂志和小报的变现模式胜利了。

这种“收集并转卖”的模式最终把我们带到了如今这个世界：社交媒体发展成了对收集到的大众数据进一步加以利用，并将其出售给了那些愿意为这些数据付费的公司。这就是我们当下吸血鬼式的互联网变现模式。

但可悲的是，这一切并非必需如此。

错误402：需要付款

这个鲜为人知的HTTP错误代码，从最初的超文本协议规范开始，就一直被保留了下来。它让我们得以一瞥，如果比特币或其它形式的电子微支付系统再早十年出现，可能会出现的情况。这个网络错误代码是网络服务器的一种响应，代表着“在发送所请求的页面之前需要先进行付款”。你看，网络的设计者很早就知道我们需要一种方法来让数据服务器获取服务报酬，但他们只是缺乏能够胜任这一工作的点对点支付网络。

因此，如今舞台已搭好，你已经看到了总体的规则以及未来的可能。下周的文章将提供更多细节，我们将详细讨论未来可能出现的情况，包括IPv6和加密生成的地址、完全安全的通信以及夺回你的数据所有权。请放心，由于比特币以BSV的形式重生了，很快这条错误代码可能真的有服务器会用上。

未来就在眼前。别走开。

在上一篇文章中，我们概述了Web 3.0的潜力，也就是为互联网提供了一个变现层级。我们还略加提及了IPv6将为这种全新的互联网模式带来的可能性。那么，冒着把牛皮吹破的风险说一句——它将改变一切。

不止于搜索

Web 3.0的实现，或者按我喜欢的叫法——“内置支付的互联网”，将是数百万人与其浏览器基本交互模式的改变，其对网络的革命意义不亚于90年代初面世的搜索引擎。回想一下，尚未使用的HTTP错误代码402表明，HTTP协议的设计者在当时已预见到了未来支付系统的发展，这一系统将允许支付信息从客户端传输到服务器，甚至无需中介就可通过互联网进行点对点传输。因此，他们看到了网络服务器需要响应来自客户端的请求，而这些客户端尚未为这些请求付款，或者还未在请求中嵌入需要付款的通知。也许还额外需要一些关于金额和具体付款方式的信息。前景已可预见，但技术尚未成熟。

快进20年

比特币被发明出来了，点对点（PTP）网络支付的问题首次被解决了。

一种网络货币的发明是必需的，它实现了迄今为止不可能的目标，付款现在可以从客户端直接发送给服务器，而不需要第三方API在中间进行支付授权。欢呼吧！

那么，在过去的十年里，我们用这项发明做了什么呢？可悲的是，根本什么也没做。但现在这一境况终于得到了改变，原始比特币协议的能力已通过BSV得以解放。这就像六分仪的发明一样，我们终于可以安全地在公海上航行了。

我们现在可以将目光投向之前遥不可及的地平线了。

比特币作为替代性的传输/网络层

我之前曾写过将比特币用作网络传输层的文章。如果你还没有读过，那就值得刷新一把技术记忆。我最近在Craig Wright博士的一段演讲中发现了对这个话题更深入的论述，他用OSI风格的图表概述了如何利用BSV来让互联网变得更加美好，并消除网络对广告的依赖。

由于比特币的交易是经过签名的，这就解决了身份认证的问题。

不再会有中间人攻击。不再会有DDoS（分布式拒绝服务）攻击。如果一个数据包没有预期发送方的签名，那么就会被丢弃。发送方可以是消息的发起人，甚至可以是可信的中间路由器。一旦建立了可信的路由，就可以强制让其它数据包也必须经过相同的路由，否则无法到达。通过在主机上的会话间使用一次性的安全通信通道，便可建立高效的VPN。互联网可以在满足你隐私需求的情况下尽可能的开放。

IPv6

随着互联网向IPv6过渡，将比特币用作网络传输协议的可能性正迅速呈指数级增长。快速回顾一下：IPv4是目前最常见的互联网地址标准类型，采用了32位的地址空间。这允许大约40亿个地址，考虑到目前联网的设备和主机的数量，剩余地址的数量已经越来越少了。IPv6引入了64位的地址空间，允许2^128个唯一地址。在可预见的未来，这已足够每个人和所有设备的使用。

IPv6使我们能够用这些独一无二的地址为每个活着的人、大小动物和互联网设备分配一个独有的标识符。不仅如此，我们还可以开始为特定的交易和事件分配地址（如从Diesel购买一条牛仔裤）。这意味着我们可以开始探索点对点支付的使用，不仅是在人与人之间，还有在设备之间——如收费终端、销售点终端、收费站和我们可能佩戴的个人设备之间使用。

我们今天面临的众多互联网安全问题和挑战都可以通过以下事实得到解决：互联网上的每一个唯一的代理或主机都可以被映射为一个直接的实体、个人或设备。再加上每个IP数据包信息都能由设备或发送者进行签名，这就保证了我们的互联网不会遭受大多数形式的黑客攻击或信息窃取。假设每一个信息包都可追溯到其发送者，可能是一个真实注册的自然人或是一台真实注册的设备。在这种情况下，就不可能有中间人（MITM）攻击，因为每一个通过网络的数据包都可以被追责和追踪。这就像让Brinks用他们自己的安全装甲车来传输你的每一条信息，只不过你甚至不必信任Brinks。这就是终极隐私。

虽然那些警惕威权主义的人会立马开始批评这个系统显然太奥威尔了，但也许真实的情况是，如果我们让透明成为可能，这个网络将能够进行自我监督。此外，腐败依赖于信息的隐藏，不可能存在于一个透明的系统中。这也可能让网络得到更好的管理，因为任何对数据的非法使用，都可以被揪出来。

最后，当今互联网的一大弊端在于，尽管我们拥有大量的加密技术，但很少有人和应用在默认使用这些技术。随着IPv6和比特币作为传输层被引入，我们将首次能够通过开放的互联网应用默认的加密通信。使用加密生成的地址（CGA）将为每笔交易提供一次性的地址，这本质上使每条通信信道都变成了互联网上的一次性路由。再配合默认的加密，我们将拥有史上隐私性最强的互联网。这只有在地址空间充足的情况下才可能实现，而且我们要有一个身份验证层（如BSV）能够以加密的方式对消息进行签名和验证。

让我们拿顾客和商户之间的商品交易标准实用范例来举个例子。

顾客将商品拿到收银台，拿出他们设备上的二维码或条形码进行扫描。利用客户提供的信息，商户的电脑就会生成一个一次性的IPv6互联网地址（比特币地址）。然后，商户的电脑将向该地址发送一条消息，里面包含了商品、付款说明和确保信息通道安全的必要信息。客户知道这个地址，因为该地址是依据他们自己设备的二维码信息所生成的，所以他们会到该地址上去查询发送至该地址的比特币交易，解密之后再用其中的信息来构建一笔给商户的支付交易。然后，客户会对该交易进行签名和加密，并通过直接信道（可能通过另一次二维码扫描或近场通信）发回给商家。由于地址事先是未知的，所以不可能对通信进行拦截。地址是一次性的，因为它是由与该交易相关的特定信息（如交易中的商品序列号）确定性生成的，而且这会带来一项额外的好处，这笔交易记录现在可被证明与现实世界的购买交易是相关联的。

设想一下，未来的会计系统将不再需要人们在单独的外部系统上追踪所有的购买行为。设想一下，区块链本身将包含费用申报、税收返还等所有必要信息。而且所有这些信息都拥有终极的隐私保护，因为只有了解信息的各方（在这个案例中就是交易的双方，即顾客和商户）才拥有解密信息所必需的密钥。因此，只有这两方之一才可以向第三方（如税务机关）报告交易的细节。这就是通过透明度带来的隐私。

这是网络隐私的终极模式。区块链记录了全球互联网的信息并承担了相应的责任。

我希望你能看到对比特币进行整合的力量，它有能力成为网络之上的支付层级，并取代大部分的互联网层级本身，从而成为在互联网上传输、路由和验证数据包和数据的协议。通过采用IPv6并将其与比特币地址和交易相结合，我们不仅能够实现隐私通信、一次性通信和100%透明的通信，而且还能保证互联网的安全，从而让当下互联网的众多漏洞不再是问题。

打开想象的格局

想象一下，网站可以根据用户的身份或他们的访问地点而采用不同的支付方式。想象一下，政府或慈善机构可以选择对互联网上的数据访问进行补贴，如此一来，他们的公民便可以免费访问通常而言要收费的网站。给信息（费用）打折从而让信息自由市场成为可能，而在一个竞争性的市场中，有价值的信息能够在虚假信息的海洋中脱颖而出。我们有可能治愈社交媒体给社会带来的瘟疫，在那些社交平台上，真相往往被能够吸引更多眼球的东西所淹没。

与此相比，谷歌在运营搜索引擎时除了广告公司的付费之外没有任何收入，而这些广告公司想要在任何可能的时候将广告展示在你的眼前。摆脱中间人，直接向搜索引擎付费，便会好得多。这甚至可以激励他们为你提供更好的搜索结果。如果你想看广告，你应该从这一行为中得到报酬，这是Web 3.0和比特币在互联网上支持的另一种收入模式。

想象一下这样的一个世界，为了让你收到一封电子邮件，发件人必须向你支付一分钱。这将是垃圾邮件的终结。这将是广告型互联网的终结。这将是旧互联网的终结——你的个人数据在此被窃取。这将是新互联网的开始——你生产的数据内容越有价值，你的收入就越多。

这是互联网真正去中介化的开始。在过去的20年里，我们经历了向信息时代的过渡。现在将是知识时代的开端。
