比特币和IPv6的共生

Craig S. Wright 博士（nChain 的创始人兼首席科学家）说，当比特币首次推出时，它提出了真正的点对点交易，这与当前的模型非常不同，后者认为交易通过大网格在点对点和节点之间发生。

在最近于蒙特利尔举行的 IEEE 第五届未来网络世界论坛上，Wright 表示这是扩展区块链的一个问题，因为地球上的每台机器都不可能联系到每个节点。他说，这也是节点很少的原因。

与IPv6 促进的点对点通信相比，这是一种不必要的低效率- 一种定义 128 位 IP 地址的互联网协议版本，可以解释几乎无限数量的独特设备，使它们可以直接通过互联网寻址。

‘区块链节点不是去中心化的。你不会通过让许多家庭用户假装运行节点来分散比特币。你通过让很多用户直接连接来分散比特币。

Wright 说，比特币最初拥有所有这些直接连接的原因是为了避免经过各种不同的网络。他说，比特币可以让个人之间直接交流。

‘比特币被设计成一个用户和另一个用户之间的电子现金。这与大多数人认为比特币的概念截然不同。

尽管节点记录交易，但它们不应该作为主要用于记录交易和处理的方法。节点对交易的顺序进行投票，并接受他们首先看到的交易。

Wright 说，当比特币被创建时，进行交易的主要方式是通过 IP 到 IP 交换。比特币是一种直接通信系统，使用户能够直接交换价值。因此，系统反映了现金。

使用 IPv6 的 IP 到 IP 事务

为了了解在不久的将来交易会是什么样子，Wright 举了一个使用 IPv6 CGA (++) 的非交互式比特币 IP 交易的例子。

此处的过程是利用 Ipv6 和 CGA 的有益特性的修改。要向不在线（非交互式）的接收者发送比特币交易，发送者可以利用 CGA++ 中使用签名的嵌入式身份验证。

要向域名而不是 IPv6 CGA 发送付款，发件人只需在过程开始时添加一个额外的步骤来解析映射到所需域名的 IP 地址。

在 CGA 参数中，有一个文本字段参数，可以在其中添加一个可选的可变长度字段（默认长度为 0）。如果接收方/服务器有域名，他们可以将其放在那里，以便客户端添加安全保证 NDS 映射是真实的。

权力下放：形成比特币网络的边缘

怀特指出，边缘计算，而不是节点网格，是去中心化的秘密。

点对点交易的本质在于参与交换或交易的各方之间的直接通信。这是因为它是电子现金，怀特重申。

节点充当诚实的代理并记录信息，并返回有关交易有效性的信息。 在这种运营形式下，节点不是金融机构。 权力下放涉及将信息推向边缘。

他说：““权力下放”一词是指个人和少数群体能够独立行动。”

关于所有权的真相

在谈到不变的所有权的价值时，怀特强调了最近的一个案例，他在亚马逊Prime上购买了一个视频系列，后来受到了审查。 怀特（Wright）指出，这是可能的，因为用户实际上并没有在许多情况下拥有他们所支付的内容。

怀特说，区块链技术可用于促进廉价、安全和透明的程序，用于专有软件的购买、完整性和许可。通过这种方式，消费者可以从持续的完整性和透明度中受益。

软件供应商可以确保在持续且具有成本效益的基础上遵守与许可证相关的条件。软件供应商还可以通过被视为透明来提高他们的声誉并增加他们的可信度。

他说，软件供应商和消费者之间不可篡改的交易记录对于审计和争议解决都可能很有价值。
